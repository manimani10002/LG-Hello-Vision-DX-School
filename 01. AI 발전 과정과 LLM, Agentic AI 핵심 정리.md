# TIL – AI 발전 과정과 LLM, Agentic AI 핵심 정리

## 1. AI의 발전 과정

### 1-1. 인지형 AI (Cognitive AI)
인지형 AI는 인간의 판단·인식·분류·예측 능력을 모방하는 것을 목표로 한다.

**학습 특징**
- 대량의 데이터를 기반으로 학습
- 명시적인 규칙이 아니라 통계적 패턴과 확률 분포를 학습
- 목적 함수(Loss)를 최소화하는 방향으로 자동 최적화

**주요 알고리즘**
- 머신러닝(Machine Learning)
- 딥러닝(Deep Learning)
- 트랜스포머(Transformer)

**AI와 데이터 분석가의 차이**
- 데이터 분석가는 가설을 세우고 데이터를 해석
- AI는 의미를 해석하려 하지 않고, 새로운 데이터를 예측 모델에 입력하여 결과를 출력
- AI의 강점은 “이해”가 아니라 “예측”

---

## 2. 생성형 AI (Generative AI)

생성형 AI는 기존 데이터를 단순히 분석하는 것이 아니라,  
확률적으로 가장 그럴듯한 다음 결과를 생성하는 AI이다.

- 텍스트, 이미지, 음성, 코드 등 다양한 결과 생성 가능
- LLM(Large Language Model)은 생성형 AI의 대표적인 예

---

## 3. LLM과 Agentic AI

### 3-1. Agentic AI 개념
Agentic AI는 LLM이 단순 응답을 넘어서 **행동할 수 있도록 확장된 구조**이다.

```

LLM + Tools = Agent

```

**Tool 예시**
- 검색 API
- 데이터베이스
- 계산기
- 코드 실행기
- RAG (Retrieval Augmented Generation)

RAG 역시 LLM이 외부 정보를 활용하기 위한 하나의 Tool이다.

---

### 3-2. LLM의 주요 한계
1. 환각(Hallucination): 그럴듯하지만 사실과 다른 정보 생성
2. 개인화 정보 부재: 내부 문서·사내 데이터 접근 불가
3. 보안 문제: 민감 데이터 외부 전송 위험

→ 이러한 이유로 LLM을 단독으로 사용하지 않고  
RAG, Fine-tuning, 온프레미스 모델, Agent 구조를 활용한다.

---

## 4. LLM 튜닝의 정확한 의미

### 4-1. Prompt Engineering
- 입력을 잘 설계하는 기법
- 모델의 파라미터는 변경되지 않음
- 튜닝(Fine-tuning)과는 다름

### 4-2. Fine-tuning
- 기존 Foundation Model 위에 추가 데이터로 가중치를 미세 조정
- 기존 지식을 덮어쓰지 않고 “+” 방식으로 보완
- Catastrophic Forgetting 방지가 핵심

---

## 5. RAG 구조

### 5-1. Vector RAG
- 의미 유사도 기반 검색
- 임베딩 + 벡터 DB + Cosine Similarity
- 자연어 검색에 강점

### 5-2. Graph RAG
- 관계 기반 탐색
- 노드-엣지 구조
- 원인·관계·흐름 질문에 강점

| 구분 | Vector RAG | Graph RAG |
|----|----|----|
| 기준 | 의미 유사도 | 관계 |
| 장점 | 유연한 검색 | 논리적 연결 |
| 단점 | 관계 이해 약함 | 구축 비용 |

---

## 6. Agent 구조 확장

- Single Agent + Tools  
  → 외부 API 및 시스템 기능 호출

- Single Agent + Tools + MCP Servers  
  → 여러 시스템을 표준화된 방식으로 연동

*MCP(Model Context Protocol)*은  
Agent와 Tool 연동을 표준화하기 위한 개념이다.

---

## 7. 트랜스포머(Transformer)

### 7-1. 구조
- Encoder / Decoder 구조
- RNN, CNN의 한계를 극복

### 7-2. 핵심 특징: Attention
- 모든 토큰 간 관계를 동시에 계산
- 거리와 무관하게 중요도 반영

자연어는 토큰 단위로 처리되며,  
토큰의 크기는 모델과 언어마다 다르다.

### 7-3. Self-Attention
- 문장 내에서 중요한 단어에 높은 가중치 부여
- 프롬프트에서 핵심 키워드를 명확히 제시하면 성능 향상

---

## 8. 생성 파라미터

### 8-1. Temperature
- 출력의 랜덤성 제어
- 0에 가까울수록 결정적인 답변
- ChatGPT 내부 값은 공식적으로 공개되지 않음

### 8-2. Top-p
- 누적 확률 p 범위 내에서 토큰 선택
- 응답 다양성 조절

---

## 9. 프롬프트 메시지 구조
1. System Message: 역할과 규칙
2. Assistant Message: 이전 응답
3. User Prompt: 사용자 요청

---

## 10. 효과적인 프롬프트 작성 원칙

- 명확성(Clarity)
- 맥락 제공(Context)
- 출력 형식 지정(Format)
- 제약 조건(Constraints)
- 예시 제공(Few-shot)
- 역할 부여(Role Prompting)

Q&A 템플릿은 지시와 입력을 포함하고  
Markdown 형식 사용이 효과적이다.

---

## 11. 멀티모달(Multimodal)

멀티모달이란 하나의 모델이 여러 입력 모달리티를 직접 이해하는 것.

- 텍스트
- 이미지
- 음성
- 비디오

Tool 호출과 멀티모달은 개념적으로 다르며,  
멀티모달은 모델 내부에서 직접 처리한다.

---

## 12. 핵심 요약
- AI는 분석가가 아니라 예측 시스템
- LLM 단독 사용은 한계가 있음
- Agent + RAG 구조가 실무 핵심
- Prompt Engineering ≠ Fine-tuning
- Foundation Model 위에 Fine-tuning으로 특화 모델 생성
- 멀티모달은 Tool 사용과 다르다
