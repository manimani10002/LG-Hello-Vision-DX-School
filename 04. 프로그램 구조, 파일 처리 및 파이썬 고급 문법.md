## 1. 프로그램과 함수의 정의

* **프로그램**: 특정 산출물을 얻기 위해 명령어(Instruction)와 데이터(Data)를 결합한 집합체.
* **함수(Function)의 등장 배경**:
* **코드 재사용**: 중복 코드를 제거하여 개발 및 유지보수 비용 절감.
* **모듈화**: 복잡한 문제를 기능 단위로 분할하여 관리 가능하게 함.



## 2. 인공지능 개발과 설계의 중요성

* 인공지능(LLM 등)은 입력값이 같아도 결과가 달라질 수 있는 **비결정론적(Non-deterministic)** 특성이 있습니다.
* 인공지능의 비일관성은 시스템의 신뢰도를 낮추므로, 이를 제어하기 위한 **프롬프트 엔지니어링, 결과 검증 로직, 파이프라인 설계** 능력이 현대 개발자에게 매우 중요

## 3. 파일 처리 시스템 (File I/O)

* **데이터 이동**: 하드디스크(보조기억장치)의 데이터를 실행을 위해 RAM(주기억장치)으로 적재하는 과정이 핵심임.
* **Stream(스트림)의 특징**:
* **단방향성**: 데이터는 한 방향으로만 흐름. (입력용 빨대와 출력용 빨대가 각각 필요)
* **프로세스 기준**: 데이터가 들어오면 `Input(Read)`, 나가면 `Output(Write)`.


* **기본 코드 패턴**:
```python
with open("./test.txt", "r") as f: # with문 권장 (자동 close)
    for line in f:
        print(line.strip())

```



## 4. 파이썬 고급 함수 문법

### (1) 가변 인자 (Packing)

* `*args`: 여러 개의 위치 인자를 **튜플(Tuple)**로 묶어서 받음.
* `**kwargs`: 여러 개의 키워드 인자를 **딕셔너리(Dict)**로 묶어서 받음.

### (2) 구조 분해 할당 (Unpacking)

리스트나 딕셔너리에 담긴 데이터를 여러 변수에 나누어 담거나, 함수의 인자로 풀어서 전달하는 기법입니다.

* **리스트 언패킹**: `a, b = [10, 20]`
* **함수 호출 시 언패킹**:
```python
def add(a, b): return a + b
data = [10, 20]
print(add(*data)) # add(10, 20)과 동일

```



### (3) 람다 함수 (Lambda Function)

이름이 없는 익명 함수로, 메모리 절약과 코드 간결화를 위해 사용됩니다. 주로 **일회성**으로 쓰이며, 다른 함수의 인자로 넘겨줄 때 유용합니다.

**문법:** `lambda 매개변수 : 표현식`

**예시 코드:**

1. **기본 호출**: `(lambda x, y: x + y)(10, 20)`  결과: 30
2. **정렬(sort)에 활용**:
```python
points = [(1, 2), (3, 1), (5, -1)]
# y좌표(index 1)를 기준으로 정렬하고 싶을 때
points.sort(key=lambda x: x[1]) 
# 결과: [(5, -1), (3, 1), (1, 2)]

```


3. **조건문 사용**:
```python
check_even = lambda x: "짝수" if x % 2 == 0 else "홀수"
print(check_even(4)) # 결과: 짝수

```

**추가**:
