## 💡 [36일차] SVM 심화 및 웹 데이터 기술

### 1. SVM (Support Vector Machine)의 철학

* **과적합(Overfitting) 방지:** 학습 데이터에만 너무 딱 맞게 경계선을 그으면 새로운 데이터(Test data)를 분류하지 못합니다. SVM은 이를 방지하기 위해 **마진을 최대화**하여 '가장 여유 있는' 결정 경계를 세웁니다.
* **초평면(Hyperplane):** 데이터가 2차원이면 경계는 선(Line)이지만, 3차원 이상에서는 면(Plane) 이상의 기하학적 형태를 띱니다. 이를 통칭하여 초평면이라 부르며, SVM은 이 평면을 찾는 **선형 모델**입니다.
* **지지 벡터(Support Vector):** 결정 경계를 만드는 데 기여하는 가장 외곽의 데이터 포인트들입니다. 이들만 알면 나머지 데이터는 없어도 경계를 유지할 수 있습니다.

### 2. 하이퍼파라미터와 커널 함수

SVM의 성능을 좌우하는 핵심 설정값들입니다.

#### ① 커널 함수의 정체 (Kernel Trick)

* **개념:** 저차원에서는 선형 분리가 불가능한 데이터를 **고차원으로 변환**하여 선형 분리가 가능하게 만드는 기법입니다.
* **내적 유지의 중요성:** 변환 시 데이터의 본질적 관계(내적)가 깨지면 안 됩니다. 이를 수학적으로 증명한 것이 **'머서의 정리(Mercer's Theorem)'**이며, 이를 만족하는 함수만 커널로 쓸 수 있습니다.
* **선형(Linear):** 변환 없이 직선으로 분리.
* **가우시안 RBF(Radial Basis Function):** 무한 차원으로 변환하는 효과, 가장 범용적임.
* **시그모이드(Sigmoid):** 로지스틱 함수와 유사한 형태.



#### ② 오차 허용도 (C)와 감마 ($\gamma$)

* **Cost (C):** 오차를 얼마나 허용할 것인가?
* 가 크면: 오차를 허용 안 함  꼼꼼하게 분류  **과적합 위험**.


* **Gamma ($\gamma$):** 가우시안 함수의 영향력 범위(폭).
* $\gamma$가 높으면: 영향력 범위가 좁아짐(폭이 좁은 종 모양)  데이터 하나하나에 민감하게 반응  **과적합 위험**.

### 3. 인공신경망 vs SVM

* **인공신경망(ANN):** 전체 데이터를 반복적으로 학습하며 가중치를 조정합니다. 블랙박스적 성격이 강하고 데이터가 많을수록 유리합니다.
* **SVM:** 경계 근처의 **지지 벡터**만 집중적으로 봅니다. 데이터가 적어도 안정적이며, 수학적으로 명확한 결론(전역 최적해)을 보장합니다.

### 4. R 실습 코드 분석

작성하신 코드에서 오타(`kenel` -> `kernel`, `rgfdot` -> `rbfdot`)를 수정하고 설명을 덧붙였습니다.

```r
# 필요한 패키지 설치 및 로드
# install.packages("kernlab")
# install.packages("caret")
library(kernlab) # ksvm 함수 제공
library(caret)   # 데이터 분할 및 평가 도구 제공

# 1. 데이터 분할 (7:3 비율)
idx <- createDataPartition(iris$Species, p=0.7, list=F)
iris_train <- iris[idx,]
iris_test  <- iris[-idx,]

# 2. SVM 모델 학습
# kernel="rbfdot": 가우시안 RBF 커널 사용
# C 값이나 kpar(gamma)를 지정하지 않으면 기본값으로 자동 최적화 수행
m <- ksvm(Species ~ ., data = iris_train, kernel = "rbfdot")

# 3. 예측
pred <- predict(m, iris_test, type = "response")

# 4. 성능 평가 (혼동 행렬)
# 정답률(Accuracy), 정밀도(Precision), 재현율(Recall) 등을 한눈에 확인
confusionMatrix(pred, iris_test$Species)

```

### 5. 웹 기술: 데이터 통신 규격

* **XML:** 태그(`< >`)를 이용해 데이터 구조를 표현하는 전통적인 방식. 사람이 읽기 좋지만 데이터 크기가 큼.
* **JSON:** Key-Value 쌍으로 이루어진 가벼운 형식. 현재 웹 표준으로 가장 많이 쓰이며 자바스크립트와 호환성이 최고임.
* **AJAX:** 비동기식 자바스크립트 통신. 페이지 전체를 새로고침 하지 않고 필요한 데이터(JSON 등)만 서버에서 받아올 때 사용함.
