# 📘 29일차 TIL: 비지도 학습(Unsupervised Learning) 완벽 정리

비지도 학습은 정답(Label)이 없는 상태에서 데이터 자체의 패턴, 구조, 특성을 찾아내는 머신러닝 방법론입니다.

---

## 1. 군집화 (Clustering)

데이터를 유사한 특징을 가진 그룹(Cluster)으로 묶는 기술입니다.

* **K-평균(K-Means):** 데이터를 K개의 중심점으로 할당하고 중심을 업데이트하며 그룹화하는 가장 대표적인 알고리즘.
* **계층적 군집화(Hierarchical Clustering):** 개별 데이터들을 가까운 것부터 순차적으로 묶어 트리 구조(Dendrogram)를 형성하는 방식.
* **DBSCAN:** 밀도(Density) 기반 군집화. 기하학적인 모양의 군집도 잘 찾아내며, 이상치(Outlier) 탐지에 강점이 있음.
* **주요 활용:** 고객 세그먼테이션, 이미지 분할, 이상 탐지.

## 2. 연관 분석 (Association Analysis)

데이터 내 항목들 간의 '함께 발생하는' 규칙을 찾아내는 기법입니다. (예: 장바구니 분석)

### 핵심 평가지표 (중요)

연관 규칙이 얼마나 유의미한지 판단하는 3가지 척도입니다.

1. **지지도 (Support):** 전체 거래 중 항목 A와 B가 **동시에 포함된** 비율.
* $P(A \cap B)$ : 규칙이 얼마나 빈번하게 발생하는가?


2. **신뢰도 (Confidence):** A를 구매했을 때 **B도 구매할 확률**.
* $P(B|A) = \frac{P(A \cap B)}{P(A)}$ : 규칙의 조건부 확률이 얼마나 높은가?


3. **향상도 (Lift):** A와 B가 독립적일 때보다 **얼마나 더 자주 함께 발생하는가?**
* $\frac{P(B|A)}{P(B)}$
* **Lift > 1:** 양의 상관관계 (의미 있는 규칙)
* **Lift = 1:** 독립 (관계없음)
* **Lift < 1:** 음의 상관관계 (서로 피하는 경향)



## 3. 추천 시스템 (Recommendation System)

사용자의 과거 행동이나 아이템의 특성을 분석하여 선호할 만한 아이템을 제안합니다.

* **콘텐츠 기반 필터링 (Content-based Filtering):** 사용자가 좋아했던 아이템과 유사한 특성(장르, 배우, 소재 등)을 가진 다른 아이템을 추천.
* **협업 필터링 (Collaborative Filtering):**
* **사용자 기반:** 나랑 취향이 비슷한 '다른 사람'이 본 것을 추천.
* **아이템 기반:** 내가 본 아이템과 '함께 팔린' 아이템을 추천.


* **행렬 분해 (Matrix Factorization):** 사용자-아이템 행렬의 빈 공간을 수학적 기법(SVD 등)으로 채워 넣어 점수를 예측.

## 4. (추가) 차원 축소 (Dimension Reduction)

강의 목차에는 없으나 비지도 학습에서 매우 중요한 파트입니다. 고차원의 데이터를 핵심 정보만 남겨 저차원으로 압축합니다.

* **PCA (주성분 분석):** 데이터의 분산을 최대한 보존하면서 서로 독립적인 새로운 축(주성분)을 찾아냄.
* **활용:** 시각화, 데이터 압축, 노이즈 제거, 지도 학습의 전처리.

## 💡 요약 및 비교

| 구분 | 목적 | 핵심 키워드 |
| --- | --- | --- |
| **군집화** | 비슷한 것끼리 묶기 | 거리(Euclidean), 중심점, 밀도 |
| **연관분석** | "A하면 B한다" 찾기 | 장바구니, 지지도/신뢰도/향상도 |
| **추천시스템** | 선호 아이템 예측 | 유사도, 협업 필터링, 행렬 분해 |
| **차원축소** | 데이터 단순화/압축 | 분산 보존, PCA, 효율성 |
