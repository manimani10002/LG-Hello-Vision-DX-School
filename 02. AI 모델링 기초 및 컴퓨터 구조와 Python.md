### 1.2. 💡 핵심 용어 정리: LLM 튜닝 및 경량화

#### 🧱 Foundation Model (기반 모델)
- Google, OpenAI 등이 방대한 데이터로 **사전 학습(Pre-training)** 한 거대 모델
- 예시: `GPT-4`, `Llama 3`, `Gemini`
- 핵심 개념:
- “기본은 웬만하면 건드리지 않는다”
- 이미 학습된 **범용 지식 유지**

#### 🔧 Full Fine-tuning (전체 파인튜닝)
- 기반 모델의 **모든 파라미터(가중치)**를 학습
- 단점:
- 비용이 매우 큼
- **Catastrophic Forgetting** (기존 지식 소실 위험)

#### 🪶 PEFT (Parameter-Efficient Fine-tuning)
- 최근 트렌드
- 기반 모델은 **Freeze**
- 소량의 파라미터만 추가 학습
- 대표 기법: `LoRA`

#### 📦 SLM (Small Language Model)
- 파라미터 수를 줄인 **소형 언어 모델**
- 온디바이스 환경에 적합

## 2. 💻 컴퓨터 구조와 S/W 작동 원리

### 2.1. 프로그램과 실행 구조
- **프로그램(S/W)**:  
하드웨어(H/W)에 특정 동작을 수행하도록 지시하는 **명령어 집합**

- **실행 계층**

Application S/W → System S/W(OS, 컴파일러) → Hardware


---

### 2.2. CPU (중앙처리장치)

#### 🔩 구성
- 제어장치(Control Unit)
- 연산장치(ALU)

#### ⚙️ 작동 특징
- 명령을 내리고(제어), 계산을 수행(연산)
- 입출력 작업 시:
- 주변장치가 작업하는 동안 CPU는 대기하거나
- 다른 작업을 수행 (**스케줄링**)

- 단일 코어:
- 한 번에 하나의 명령어 처리
- 매우 빠른 속도로 번갈아 실행하여 **동시처럼 보이게 함**

#### 🧵 스레드(Thread)
- **개념**: 프로세스 내부에서 실제 작업을 수행하는 실행 흐름 단위
- **역할**: 하나의 프로그램에서 여러 작업을 동시에 처리하는 것처럼 보이게 함

---

### 2.3. 메모리 계층 구조 (Memory Hierarchy)

#### 🧠 주기억장치 (Main Memory)

##### RAM (Random Access Memory)
- 역할: CPU가 현재 사용하는 데이터와 명령어 저장
- 특징:
- 빠름
- **휘발성**
- 우리가 흔히 말하는 “메모리”

##### ROM (Read Only Memory)
- 역할: 읽기 전용 메모리
- 특징:
- **비휘발성**
- BIOS / UEFI 등 부팅 관련 핵심 정보 저장

#### 💾 보조기억장치 (Secondary Memory)
- HDD, SSD
- 역할: 데이터를 파일 형태로 **영구 저장**
- 특징:
- 비휘발성
- 속도가 느려 CPU가 직접 접근하지 않음

#### 🔄 Load (로드)
- 보조기억장치의 프로그램/데이터를
- **RAM으로 복사**하는 과정

## 3. 🛠️ Python 개발 환경의 이해

Python은 하드웨어를 직접 제어하기 어렵기 때문에  
다양한 **SDK, 라이브러리**를 활용함

### 3.1. 개발 도구 3대장

| 도구 | 역할 비유 | 설명 |
|----|----|----|
| Anaconda | 공구함 관리자 | Python 실행 파일과 라이브러리를 관리하고 가상환경 제공 |
| Jupyter Notebook | 연습장 | 코드 한 줄씩 실행, 결과 즉시 확인 |
| VS Code | 작업대(IDE) | 실제 프로젝트 개발 및 관리 |

#### 🔄 실제 작동 원리

##### Anaconda (엔진 공급소)
- 특정 버전의 `python.exe` 와 라이브러리가 포함된 **가상환경 생성**
- 예:
```
C:\Users...\anaconda3\envs\my_project
```

##### VS Code / Jupyter (조종석)
- 직접 코드를 해석하지 못하는 **Interface**
- 실행 시:
- 선택된 가상환경의 `python.exe`에게 코드 전달
- 실행 결과를 받아 화면에 출력

##### 관계 요약
- Anaconda = 엔진 & 부품 관리소
- VS Code / Jupyter = 핸들 & 계기판
- **Select Interpreter** 설정 필수

#### 🧪 가상환경 (Virtual Environment)
- 프로젝트별 라이브러리 충돌 방지를 위한 **격리된 공간**

#### 📄 파일 확장자
- `.py` : 파이썬 스크립트 (전체 실행)
- `.ipynb` : 주피터 노트북 (셀 단위 실행 + 결과 저장)

---

### 3.2. 라이브러리 vs 프레임워크
- **라이브러리**:  
내가 필요할 때 호출 → 제어권이 나에게 있음
- **프레임워크**:  
정해진 틀에 코드 작성 → 제어권이 프레임워크에 있음

### 4.1. 프로그램의 구성

### 4.2. 변수(Variable)

변수명 ↔ 주소(Address) ↔ 값(Value)

- 변수명은 메모리 주소를 대신하는 **이름표**

### 4.4. 리스트(List)와 메모리 구조
- 여러 데이터를 **순서대로 저장**
- 메모리에는 실제 값이 아닌
- **데이터가 저장된 주소(Reference)**가 연속적으로 저장됨
- 장점:
- 서로 다른 타입 데이터 혼합 가능

#### 🔢 인덱싱(Indexing)
- 0부터 시작하는 순번을 이용해 데이터 접근
