## 💡 [42일차] 효율적인 파인튜닝과 이미지 세그멘테이션

### 1. 딥러닝 파인튜닝의 진화: PEFT

기존 파인튜닝은 모델이 커질수록 메모리 비용이 감당하기 힘들다는 단점이 있었습니다. 이를 해결하기 위해 **PEFT(Parameter-Efficient Fine-Tuning)** 기술이 등장했습니다.

* **LoRA (Low-Rank Adaptation):** 기존 모델의 무거운 가중치()는 그대로 얼려두고(Freeze), 옆에 아주 작은 두 개의 행렬()만 추가하여 이 부분만 학습시킵니다. 학습 파라미터가 1% 미만으로 줄어들면서도 성능은 유지됩니다.
* **QLoRA (Quantized LoRA):** 모델의 가중치를 4비트 등으로 **양자화(압축)**하여 메모리 사용량을 극단적으로 줄인 기술입니다. 덕분에 일반 소비자용 GPU에서도 거대 모델을 튜닝할 수 있게 되었습니다.

---

### 2. 이미지 분석의 단계 (Single vs Multi Object)

* **Classification + Localization:** 이미지에 사물이 '하나'일 때, 무엇인지 맞히고 박스(Bbox)를 칩니다.
* **Object Detection:** 여러 개의 사물을 각각 박스로 찾아냅니다.
* **Instance Segmentation:** 박스를 넘어 사물의 **외곽선(픽셀 단위)**까지 정교하게 분리해냅니다.

---

### 3. 세그멘테이션의 핵심: 인코더-디코더 구조

실습하신 코드는 전형적인 **U-Net** 스타일의 구조를 띠고 있습니다.

* **인코더 (Encoder / 축소 경로):** `Conv2D`와 `strides=2`를 사용해 이미지의 공간 정보를 압축하고 중요한 특징(Feature)만 남깁니다.
* *질문 답변:* `strides=2`는 필터가 두 칸씩 이동하므로 결과물의 가로세로 크기가 절반으로 줄어듭니다. 즉, 정보의 위치 정보를 요약하는 **Pooling과 같은 역할**을 수행합니다.


* **디코더 (Decoder / 확장 경로):** 압축된 특징 지도를 다시 원본 이미지 크기로 키우는 과정입니다. 이때 `Conv2DTranspose`(Up-sampling)를 사용합니다.

---

### 4. 코드 심화 분석 (Padding & Transpose)

#### 📝 Padding="same"의 차이 (중요!)

코드에서 `Conv2D`와 `Conv2DTranspose` 모두 `padding="same"`을 썼지만, 그 역할은 반대 방향입니다.

1. **Conv2D (Down-sampling):** * 입력 이미지를 훑을 때 테두리를 0으로 채워, **출력 크기가 (입력 크기 / 스트라이드)**가 되도록 유지합니다.
2. **Conv2DTranspose (Up-sampling):** * 줄어든 데이터를 다시 키울 때, **정확히 (입력 크기  스트라이드)**의 크기가 되도록 빈 공간을 계산하여 맞춥니다.
* 즉, 인코더에서 줄어든 비율만큼 디코더에서 정확히 대칭으로 복원하기 위해 사용하는 설정입니다.

### 5. 세그멘테이션 실습 코드 분석

```python
def get_model(img_size, num_classes):
    inputs = keras.Input(shape=img_size + (3,))
    
    # [데이터 정규화] 0~255 픽셀을 0~1 사이로 변환
    x = layers.Rescaling(1./255)(inputs)

    # === [인코더 구간] 특징 추출 및 압축 ===
    # strides=2를 통해 이미지 크기를 1/2, 1/4, 1/8로 계속 줄여나감
    x = layers.Conv2D(64, 3, strides=2, activation="relu", padding="same")(x)
    x = layers.Conv2D(64, 3, activation="relu", padding="same")(x) # 크기 유지하며 특징 정교화
    x = layers.Conv2D(128, 3, strides=2, activation="relu", padding="same")(x)
    x = layers.Conv2D(128, 3, activation="relu", padding="same")(x)
    x = layers.Conv2D(256, 3, strides=2, padding="same", activation="relu")(x)
    x = layers.Conv2D(256, 3, activation="relu", padding="same")(x)

    # === [디코더 구간] 압축된 정보를 원본 크기로 복원 ===
    # Conv2DTranspose와 strides=2를 사용해 크기를 다시 2배씩 키움
    x = layers.Conv2DTranspose(256, 3, activation="relu", padding="same")(x)
    x = layers.Conv2DTranspose(256, 3, activation="relu", padding="same", strides=2)(x)
    x = layers.Conv2DTranspose(128, 3, activation="relu", padding="same")(x)
    x = layers.Conv2DTranspose(128, 3, activation="relu", padding="same", strides=2)(x)
    x = layers.Conv2DTranspose(64, 3, activation="relu", padding="same")(x)
    x = layers.Conv2DTranspose(64, 3, activation="relu", padding="same", strides=2)(x)

    # === [출력층] ===
    # 마지막은 분류하려는 클래스 수(num_classes)만큼의 채널을 가짐
    # 각 픽셀이 어떤 클래스에 속하는지 확률(softmax)로 결정
    outputs = layers.Conv2D(num_classes, 3, activation="softmax", padding="same")(x)

    model = keras.Model(inputs, outputs)
    return model

```
