# 📘 34일차 TIL: 웹 프로토콜과 머신러닝 모델의 웹 서비스화

### 1. 웹 네트워크 프로토콜 계층

네트워크는 계층별로 역할이 나누어져 있습니다. 데이터가 오고 가는 길을 이해하는 것이 웹 서비스의 기본입니다.

* **HTTP (Application Layer):** 브라우저와 서버가 통신할 때 사용하는 '규약'입니다. 텍스트, 이미지 등을 주고받기 위한 최상위 계층입니다.
* **TCP/IP (Transport/Network Layer):** 운영체제(OS) 내부에 구현되어 있으며, 데이터를 작은 패킷 단위로 쪼개고 목적지까지 안전하게 전달하는 역할을 합니다.
* **MAC (Data Link Layer):** 하드웨어(랜카드)의 고유 주소입니다. 물리적인 장치끼리 데이터를 식별할 때 사용합니다.

---

### 2. AJAX (Asynchronous JavaScript and XML)

웹 페이지 전체를 새로고침하지 않고도, **페이지의 일부분만 데이터를 업데이트**하는 기술입니다.

* **추천 시스템에서의 역할:** 사용자가 '좋아요'를 누르면 화면 전환 없이 실시간으로 추천 목록이 바뀌는 기능 등을 구현할 때 필수적입니다.
* 최근에는 XML 대신 **JSON(JavaScript Object Object Notation)** 형식을 주로 사용합니다.

---

### 3. 웹 패키징 및 서비스 절차 (ML 모델 배포)

학습시킨 모델을 웹에서 돌아가게 만드는 과정입니다. 용어와 흐름을 조금 더 정교하게 다듬어 보았습니다.

#### ① 모델 직렬화 (`.pkl` 파일)

* **Pickle:** 파이썬 객체(학습된 모델)를 파일 형태로 저장하는 라이브러리입니다.
* 메모리에 있는 모델을 `.pkl` 형태로 하드디스크에 저장해야 나중에 웹 서버에서 다시 불러와(Load) 예측에 사용할 수 있습니다.

#### ② 웹 애플리케이션 작성 (`app.py`)

* **Flask**나 **FastAPI** 같은 파이썬 웹 프레임워크를 사용합니다.
* 사용자로부터 데이터를 입력받아 저장된 `.pkl` 모델에 전달하고, 예측 결과()를 반환하는 로직을 작성합니다.

#### ③ 서버 프로세스 구동 (Daemon Process)

* 파이썬 스크립트가 단순히 한 번 실행되고 꺼지는 것이 아니라, **데몬(Daemon) 형태**로 백그라운드에서 계속 살아있어야 합니다.
* 그래야 사용자가 언제 접속하더라도 요청을 처리할 수 있습니다.

#### ④ 클라이언트 접속 (Web Browser)

* 사용자가 브라우저를 통해 주소(URL)를 입력하면, 구동 중인 서버에 요청이 전달되고 모델의 예측 결과가 화면에 출력됩니다.

---

### 4. 용어 수정 및 추가 팁

* **Web Packaging:** 보통은 '배포(Deployment)' 또는 '서빙(Serving)'이라는 용어를 더 많이 사용합니다.
* **데몬 프로세스:** 리눅스나 서버 환경에서 `gunicorn`이나 `uWSGI` 같은 도구를 사용해 파이썬 웹 앱을 안정적으로 구동시키는 것을 의미합니다.
